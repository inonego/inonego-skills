# 소프트웨어 개발 — 세부 설계 수준 추가 가이드

범용 [detail-scope.md](../../../references/scopes/detail-scope.md)의 각 단계에 **추가**로 적용되는 소프트웨어 전용 항목.

---

### 설계 방향 추가 항목

| 초안 항목 | 설명 |
|-----------|------|
| 기존 코드와의 관계 | 영향 받는 기존 코드, 호환성 고려사항 |
| 기술적 제약 | 플랫폼, 언어/프레임워크 버전, 런타임 제한 |
| 영향 범위 | 변경이 미치는 모듈/서비스 목록 |

> "기존 코드베이스의 컨벤션/패턴과 어떻게 조화시킬 것인가?"
> "이 변경으로 영향 받는 기존 기능이 있는가?"

### 구조 설계 추가 항목

| 초안 항목 | 설명 |
|-----------|------|
| 설계 원칙 | 적용할 원칙과 근거 (SOLID, DRY 등 맥락에 맞게 선택) |
| 인터페이스 | 공개 API, 시그니처, 데이터 구조 |
| 결합도 관리 | 구성요소 간 의존성을 낮추는 방식 |

> "데이터 흐름 방향이 적절한가? (단방향 vs 양방향)"
> "동시성/스레드 안전성이 필요한 구성요소가 있는가?"

### 실행 명세 추가 항목

| 초안 항목 | 설명 |
|-----------|------|
| 클래스/메서드 설계 | 시그니처, 접근 제어, 반환 타입 |
| 코드 패턴 | 구체적 구현 패턴 (콜백, 이벤트, 팩토리 등) |
| 에러 처리 | 예외/에러 전파 방식 |
| 테스트 전략 | 검증 수준, 테스트 경계, 핵심 테스트 케이스 |
| 수용 기준 | "완료"로 판단하는 조건 |

> "시그니처와 데이터 구조가 명확하고 완전합니까?"
> "에러/예외 시 호출자에게 어떻게 알릴 것인가?"
> "테스트 격리를 위해 어떤 경계를 두는가?"
