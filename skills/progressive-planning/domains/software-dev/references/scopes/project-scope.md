# 소프트웨어 개발 — 프로젝트 수준 추가 가이드

범용 [project-scope.md](../../../references/scopes/project-scope.md)의 각 단계에 **추가**로 적용되는 소프트웨어 전용 항목.

---

### 큰 그림 추가 항목

| 초안 항목 | 설명 |
|-----------|------|
| 기술 스택 | 선택한 기술과 그 근거 |
| 아키텍처 방향 | 전체 구조 패턴 (모노리스, 마이크로서비스, 모듈러 등) |
| 배포 환경 | 인프라/배포 개요 |

> "기존 시스템과 통합해야 하는 부분이 있습니까?"
> "이 시스템의 가장 중요한 품질 속성은 무엇입니까? (예: 성능, 확장성, 유지보수성)"

### 구성요소 추가 항목

| 초안 항목 | 설명 |
|-----------|------|
| 모듈/서비스 경계 | 각 모듈의 책임과 분리 기준 |
| API/데이터 흐름 | 모듈 간 통신 방식과 데이터 이동 경로 |
| 외부 연동 | 외부 시스템/서비스 연동 지점 |

> "컴포넌트 간 인터페이스/경계가 명확합니까?"
> "재사용 가능한 기존 코드가 있습니까?"

### 세부 명세 추가 항목

| 초안 항목 | 설명 |
|-----------|------|
| 기술 명세 | 파일/클래스, 인터페이스, 데이터 구조 |
| 의존성 관계 | 모듈 간 의존 방향과 관리 방식 |
| 비기능 요구사항 | 성능, 보안, 확장성 등 구체적 수치 |

> "API 시그니처와 데이터 모델이 명확합니까?"
> "성능 요구사항이 있다면 구체적인 수치는?"
