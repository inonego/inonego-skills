# 세부 설계 수준 스코프 가이드

특정 기능, 영역, 또는 구성요소의 내부 설계에 적용한다.

> 도메인 가이드가 로드되어 있으면 도메인별 스코프 가이드(`domains/도메인/references/scopes/detail-scope.md`)의 추가 초안 항목·질문을 각 레이어에서 함께 반영한다.

---

## 진행 흐름

```
설계 방향 → 구조 설계 → 실행 명세
```

### 설계 방향 — "왜 이 접근법인가"

| 초안 항목 | 설명 |
|-----------|------|
| 목적 | 이 설계가 해결하는 문제 또는 달성하려는 것 |
| 설계 의도 | 핵심적으로 지키고자 하는 원칙 |
| 접근법 후보 | 후보별 장단점 |
| 선택한 접근법 | 최종 선택과 그 이유 |

> "제시된 접근법 후보 중 선호하는 방향이 있습니까? 또는 다른 접근법이 있습니까?"

확정 후 → 핵심 구성 영역을 **하나씩** 구조 설계로 전개

### 구조 설계 — "어떤 구조로 만드는가"

| 초안 항목 | 설명 |
|-----------|------|
| 핵심 구성요소 | 구성요소와 각각의 책임 |
| 상호작용 | 구성요소 간 연결과 흐름 |
| 연결 규약 | 구성요소 간 주고받는 것과 방식 |
| 접근법 반영 | 선택한 접근법이 구조에 어떻게 나타나는지 |

> "구성요소 간 책임 분리가 적절합니까? 경계를 다르게 나누고 싶은 부분이 있습니까?"

확정 후 → 각 구성요소를 **하나씩** 실행 명세로 전개

### 실행 명세 — "구체적으로 어떻게 실행하는가"

| 초안 항목 | 설명 |
|-----------|------|
| 실행 항목 | 각 항목의 투입/산출/완료 기준 |
| 실행 방식 | 구체적 절차와 순서 |
| 예외 상황 | 예상치 못한 상황과 대응 방식 |
| 활용 예시 | 이 구성요소를 활용하는 시나리오 |

하위 계획 위임 대상이 있으면 경계만 정의 (SKILL.md "하위 계획 위임" 참조).

> "실행 항목이 명확하고 완전합니까? 빠진 항목이 있습니까?"
