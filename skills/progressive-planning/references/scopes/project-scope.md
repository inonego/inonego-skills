# 프로젝트 수준 스코프 가이드

전체 프로젝트 또는 여러 영역에 걸친 계획에 적용한다.

> 도메인 가이드가 로드되어 있으면 도메인별 스코프 가이드(`domains/도메인/references/scopes/project-scope.md`)의 추가 초안 항목·질문을 각 레이어에서 함께 반영한다.

---

## 진행 흐름

```
큰 그림 → 구성요소 → 세부 명세
```

### 큰 그림 — "무엇을 하는가"

| 초안 항목 | 설명 |
|-----------|------|
| 목표 | 한 문장으로 정리한 최종 목표 |
| 주요 단계 | 각각 한 줄 설명. 순서는 의존성 반영 |
| 범위 | 포함/제외 명시 |
| 선행 조건 | 시작 전에 필요한 것 |

> "단계 구성이 이 방향이 맞습니까? 빠진 단계나 불필요한 단계가 있으면 알려주세요."

확정 후 → 각 단계를 **하나씩** 구성요소로 분해

### 구성요소 — "어떻게 하는가"

| 초안 항목 | 설명 |
|-----------|------|
| 목표 | 이 단계가 달성해야 하는 것 |
| 구성요소 | 각각 역할과 산출물 |
| 의존성 | 구성요소 간 순서 관계 |
| 리스크 | 식별된 리스크와 대응 방안 |

> "이 분해가 적절합니까? 빠진 구성요소나 합쳐야 할 구성요소가 있습니까?"

확정 후 → 각 구성요소를 **하나씩** 세부 명세로 전개

### 세부 명세 — "정확히 무엇을 해야 하는가"

| 초안 항목 | 설명 |
|-----------|------|
| 작업 항목 | 각각 투입/산출/완료 기준 |
| 제약사항 | 지켜야 할 제약 |
| 열린 질문 | 아직 결정되지 않은 사항 |

하위 계획 위임 대상이 있으면 경계만 정의 (SKILL.md "하위 계획 위임" 참조).

> "작업 항목이 명확하고 실행 가능합니까? 애매한 부분이나 추가 분해가 필요한 항목이 있습니까?"
